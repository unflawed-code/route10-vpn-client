# Supply Chain Verification

This repo uses the official SLSA GitHub Generator for release artifacts produced by the `SLSA Provenance` workflow:

- The release workflow checks shell syntax before packaging.
- Tags matching `v*` build a release tarball from the exact Git commit.
- Release artifacts include `SHA256SUMS`.
- `slsa-framework/slsa-github-generator` generates signed SLSA Build L3 provenance for the release tarball.

The badge means release artifacts have SLSA Build L3 provenance generated by the official reusable workflow. It is not a dynamic certification badge or repo-wide conformance claim; the proof is the generated `.intoto.jsonl` provenance that users verify.

For the SLSA model and levels, see [slsa.dev](https://slsa.dev/).

## Release Artifacts

For tag `vX.Y.Z`, the release workflow publishes:

```text
route10-vpn-client-vX.Y.Z.tar.gz
SHA256SUMS
*.intoto.jsonl
```

## Verify A Release

Install the GitHub CLI and `slsa-verifier`, then download and verify:

```bash
gh release download vX.Y.Z --repo unflawed-code/route10-vpn-client
sha256sum -c SHA256SUMS
slsa-verifier verify-artifact route10-vpn-client-vX.Y.Z.tar.gz \
  --provenance-path *.intoto.jsonl \
  --source-uri github.com/unflawed-code/route10-vpn-client \
  --source-tag vX.Y.Z
```

The verifier should confirm that the artifact was produced by `github.com/unflawed-code/route10-vpn-client` from the expected tag.
